Figure 2. Document ingestion pipeline architecture illustrating the four-stage transformation process: (1) PDF document loading with metadata extraction using PyMuPDF, (2) recursive text chunking with configurable size and overlap parameters, (3) dense vector embedding generation via OpenAI's text-embedding-3-large model (d=3072), and (4) HNSW-indexed persistent storage in ChromaDB. The pipeline transforms n=45 source documents into m=445 searchable vector embeddings with O(log n) retrieval complexity.
